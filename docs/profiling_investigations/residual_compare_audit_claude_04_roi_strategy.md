## Test Assessment

Coverage for correctness of `content_generation` advancement is solid — `tests/screen_operations/model_ops_tests.cpp:3102-3207` exercises wide cells, wide-continuation overwrite, combining marks, variation sequences, no-autowrap clipped writes, style-only no-op rewrite, hyperlink-only no-op rewrite, and erase-character idempotency. `tests/render_snapshot/render_snapshot_tests.cpp` covers validation of `visible_line_provenance` and snapshot structure. `tests/qsg_render/qsg_render_tests.cpp` covers downstream `text_content_reused` / `text_clean_reuse_skips` behavior.

What's missing for the next ROI slice:
- **Materialization-cost contract**: no test asserts that emitted `snapshot.cells` count tracks dirty-row count rather than visible-row count. Without this gate, the high-ROI change in finding #1 is invisible to CI.
- **End-to-end clean-row reuse**: no direct test that proves a renderer cache hit happens when a row is unchanged AND its cells are omitted from `snapshot.cells` (today they are present, so the cache pathway has never been exercised without them).
- **Non-ASCII parity for span-local equivalence**: `test_retained_line_content_generation_mutations` validates correctness but does not pin call counts or per-call cost shape, so applying the span-local pattern to `install_cell_span` will need a small fixture that asserts `printable_ascii_local_cells_inspected` vs `row_content_generation_comparison_cells` ratios shift as expected.
- **`viewport_row_cells` direct coverage**: no test exists for this entry point today (only transitive through `render_snapshot`), so changing its return type carries hidden risk for the alternate-buffer branch (line 7075) that should be pinned before refactoring.
